### Decisions / deviations (worth noting)
1. **PushResult union kept non-flattened** per the NORMATIVE section: `{ ok: true; version: number } |
   { ok: false; conflict: true }`. I dropped the `updatedAt` field that LLD §B showed on the success
   arm — the task prompt's union (and 04-lld NORMATIVE) omit it, and `BoardRow.updated_at` is still
   available via `pullBoard` if needed.
2. **First INSERT version = 1** (per task prompt: `version: 1`, return `{ok:true,version:1}`). NOTE: LLD
   §B's pseudocode used `version: 0`. Followed the task prompt (authoritative) — INSERT writes
   `version: 1` and returns `{ok:true,version:1}`. Guarded UPDATE writes `expectedVersion + 1`.
3. **No `error.code === "23505"` unique-violation→conflict mapping on INSERT.** Task prompt's pushBoard
   spec says "return `{ok:true,version:1}`" on insert and "Surface non-conflict errors by throwing"; it
   does not ask for the 23505 mapping (that was LLD §B/§E engine-resilience detail). Insert errors
   currently throw. Flagging in case the engine task (T-engine) wants the 23505→conflict refinement
   later; trivial to add without changing the signature.
4. **`as const satisfies ReadonlyArray<keyof AppState>` (LLD §D verbatim) does NOT compile under this
   repo's lint.** The repo pins **prettier 2.6.2** and runs it as an eslint rule (`eslint-plugin-prettier`,
   `test:code --max-warnings=0`); that parser throws `SyntaxError: ',' expected` on `as const satisfies`.
   Replaced with `as const` PLUS a separate type-level assertion
   (`const _ephemeralKeysAreAppStateKeys: ReadonlyArray<keyof AppState> = EPHEMERAL_APPSTATE_KEYS;`)
   which gives the identical compile-time guarantee. Verified the guard bites: injecting a bogus key
   produces `error TS2322 ... "notARealAppStateKey" is not assignable to type 'keyof AppState'`.
5. **No ephemeral key dropped.** All 21 keys cross-checked against `APP_STATE_STORAGE_CONF`
   (`packages/excalidraw/appState.ts:149-257`) — every one is a real `browser:true` AppState key
   (activeTool:181, preferredSelectionTool:182, penMode:183, penDetected:184, zenModeEnabled:241,
   objectsSnapModeEnabled:248, scrollX/Y:220-221, zoom:242, selection/menu keys:213-245, etc.).
6. **Import-type names confirmed real** (matched `LocalData.ts`): `OrderedExcalidrawElement` from
   `@excalidraw/element/types` (def `types.ts:227`), `AppState` from `@excalidraw/excalidraw/types`,
   `SupabaseClient` (type) from `@supabase/supabase-js`, `getNonDeletedElements` from
   `@excalidraw/element`, `clearAppStateForLocalStorage` from `@excalidraw/excalidraw/appState`.
7. **Deleted-element filter** uses `getNonDeletedElements(elements)` (the same helper
   `saveDataStateToLocalStorage` uses, `LocalData.ts:92`), spread into a fresh array so the
   `readonly NonDeleted<T>[]` return assigns to `SerializedScene.document: OrderedExcalidrawElement[]`.
8. **Test mock** is an inlined adaptation of the LLD §F chainable builder (the shared
