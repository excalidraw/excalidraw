**Full-app render vs fallback:** FELL BACK to component-level (sanctioned by the task's fallback clause) — **NOT for harness heaviness but because of a real production bug** (below). Section A still exercises the actual collab entry point (`getCollaborationLinkData` is the sole producer of room data for `<Collab>.startCollaboration`), and Section B renders the real replacement UI. Standalone `<AppMainMenu>` render was rejected as fallback (iii) because `MainMenu` needs the full editor context (renders nothing standalone), which hits the same boot bug.

**BUG FOUND (release-blocking, NOT fixed — flagged per instructions):**
With `VITE_APP_FEATURE_SUPABASE_SYNC="true"`, a full `render(<ExcalidrawApp/>)` hangs on the loading spinner forever (`window.h.state.isLoading` never flips false), so the app is unusable when the flag is on (in a normal, non-iframe context).
- Root cause: the scene-init effect at `excalidraw-app/App.tsx:553-556` early-returns when `(!isCollabDisabled && !collabAPI)`. Under the flag, `<Collab>` is unmounted (`App.tsx:1111`, gated by `!isSupabaseSyncEnabled()`), and `collabAPIAtom` is set ONLY inside `<Collab>` (`collab/Collab.tsx:243`) → `collabAPI` is permanently `null`. `isCollabDisabled = isRunningInIframe()` is `false` in a normal browser/test. So `(true && true)` → effect returns early → `initialStatePromiseRef` never resolves → `isLoading` stays `true`.
- Why collab.test.tsx doesn't catch it: it runs flag-OFF, so `<Collab>` mounts, `collabAPI` becomes non-null, and the effect runs.
- Suggested fix (for the orchestrator / App.tsx owner, NOT applied here): relax the guard to also proceed under the flag, e.g.
  `if (!excalidrawAPI || (!isCollabDisabled && !collabAPI && !isSupabaseSyncEnabled())) return;`
  After that fix, the preferred full-app `render(<ExcalidrawApp/>)` disable test (assert `.collab-button` absent + `SyncStatusButton` present) should be added back.

**Stable selectors used:**
- collab trigger ABSENCE: `.collab-button` class (`LiveCollaborationTrigger.tsx:31`) + `aria-label`/`title` `"Live collaboration..."` (`labels.liveCollaboration`).
