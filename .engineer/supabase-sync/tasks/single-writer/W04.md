# W04 — UI: reader/writer button states, Take-over, read-only banner, App viewMode

## Description
Surface the role in the UI: SyncStatusButton gets reader vs writer states + a "Take over editing" affordance;
add a read-only banner; wire `viewModeEnabled` into the Excalidraw component for readers.

## Files
- `excalidraw-app/components/SyncStatusButton.tsx` (+ `.scss`) — extend
- `excalidraw-app/components/ReadOnlyBanner.tsx` (+ `.scss`) (NEW) — the read-only banner
- `excalidraw-app/App.tsx` — pass `viewModeEnabled` + render the banner

## Implement (per §3, §4, §5 SyncStatusButton.tsx / App.tsx, and §2.5 UI mockup conventions)
- **SyncStatusButton:** accept `role: SyncRole` and lock/holder info (props or read lockAtom — match how W03
  exposes it; prefer props for testability, consistent with current presentational design). States:
  - writer → existing synced/syncing/error/offline pill (label "Editing" or keep status labels; per design).
  - reader → a distinct "Read-only" pill state + a **"Take over editing"** action (button or popover item) that
    calls the `takeOver` handler passed from App. While a takeover is pending, show "Requesting…"/disabled.
  Keep it presentational (no engine import).
- **ReadOnlyBanner:** a small banner shown when `role==="reader"`: "Read-only — editing is active on another
  session" + a "Take over editing" button. Style consistent with the app (reuse theme vars like the other
  scss). Only rendered under the flag + reader role.
- **App.tsx:** pass `viewModeEnabled={supabaseSync.viewModeEnabled}` (from W03) to `<Excalidraw>` (insert near
  the existing renderTopRightUI/Excalidraw props per §5 App.tsx anchor). Render `<ReadOnlyBanner>` when
  `isSupabaseSyncEnabled() && role === "reader"`. Wire `onTakeOver={supabaseSync.takeOver}` into the button +
  banner. All flag-gated; flag-off unchanged.

### SyncStatusButton prop API (reader/writer)
New props are all **optional + backward-compatible** (existing `disable.test.tsx` renders it without them):
- `role?: SyncRole` (imported as a type from `../data/supabase/lockAtom`), defaults to `"writer"`.
- `onTakeOver?: () => void`.
- `takeoverInFlight?: boolean` (defaults `false`).
No engine/hook import — purely presentational (props in, callbacks out). Chose discrete props over passing
the whole `LockState` for testability/minimal surface (only the 3 bits the button needs).

Behavior:
- **writer** (default): unchanged synced/syncing/error/offline pill; popover keeps "Sync now".
- **reader** (`isSignedIn && role==="reader"`): distinct muted pill — glyph 👁 + label "Read-only"
  (`__pill--reader`); spinner suppressed; popover line reads "Editing is active on another session" and
  shows a **"Take over editing"** menuitem → `onTakeOver()`; while `takeoverInFlight` it shows
  "Requesting…" + `disabled`. The error row is hidden for readers (status is orthogonal to role).
- **signed-out**: unchanged "Sign in to sync".

### ReadOnlyBanner
Props `{ onTakeOver: () => void; takeoverInFlight?: boolean; writerLabel?: string | null }`. Renders
"Read-only — editing is active on <writerLabel|another session>" + a "Take over editing" button
(disabled/"Requesting…" while in flight). `role="status"`. Top-center placement mirrors the app's `.alert`
(absolute, `top: 6.5rem`, centered, `z-index: 6`); container `pointer-events: none` so it never blocks the
toolbar, button re-enables `pointer-events: auto`. Theme vars only (`--color-warning`, `--color-text-warning`,
`--color-primary`, `--color-icon-white`, `--border-radius-md`).
