`boardRepository` wrappers for them, with unit tests.

### Wrapper signatures (followed design §5, which is authoritative over the W01.md sketch)
The W01 description's sketch listed `(client, userId, sessionId, leaseSeconds, ...)`. Design §5 is the
authoritative spec and lists `(client, sessionId)` — the RPCs read `auth.uid()` server-side (no userId
arg) and use the SQL default lease/TTL (so no `leaseSeconds`/`requestTtlSeconds` args). I followed §5:

- `claimLock(client: SupabaseClient, sessionId: string): Promise<ClaimLockResult>`
  → `{ acquired, holder: { writerId, writerSessionId }, lockExpiresAt, version }`
- `renewLock(client: SupabaseClient, sessionId: string): Promise<RenewLockResult>`
  → `{ stillWriter, takeoverRequestedBy, version }`
- `releaseLock(client: SupabaseClient, sessionId: string): Promise<void>`
- `requestTakeover(client: SupabaseClient, sessionId: string): Promise<RequestTakeoverResult>`
  → `{ immediatelyClaimable, writerSessionId, lockExpiresAt }`
- `readLockState(client: SupabaseClient): Promise<LockStateResult>`
  → `{ version, writerId, writerSessionId, lockLive, takeoverRequestedBy, serverNow }`

### RPC param names used (match the SQL `create function` arg names EXACTLY)
- `client.rpc("claim_board_lock", { p_session_id })`
- `client.rpc("renew_board_lock", { p_session_id })`
- `client.rpc("release_board_lock", { p_session_id })`
- `client.rpc("request_takeover", { p_session_id })`
- `client.rpc("read_lock_state")`  (no params; `p_request_ttl_seconds` uses its SQL default of 12)

`returns table (...)` RPCs resolve to an array of rows → wrappers map `Array.isArray(data) ? data[0] :
data`; column→camelCase mapping (`lock_live`→`lockLive`, `still_writer`→`stillWriter`,
`takeover_requested_by`→`takeoverRequestedBy`, `server_now`→`serverNow`, etc.). Mutating wrappers
throw on `error` (matching `pullBoard`/`pushBoard`).
