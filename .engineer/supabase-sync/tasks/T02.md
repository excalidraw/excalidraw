# T02 — SQL migration + .env.example + setup README

## Description
Create the database migration (boards table + RLS + Storage bucket + policies) and the env/setup docs so the
stakeholder can provision their hosted Supabase project. Independent of all code tasks.

## Files
- `supabase/migrations/0001_init_boards.sql` — copy the FULL DDL from `04-lld-part1-data.md` §A verbatim:
  `boards` table (uuid pk, user_id FK→auth.users on delete cascade, unique(user_id), name, document jsonb,
  app_state jsonb, version int, created_at, updated_at), the updated_at trigger (DISPLAY ONLY — note in a SQL
  comment that LWW uses `version`, not this), RLS enable + 4 policies (select/insert/update/delete with
  `auth.uid() = user_id`), `storage.buckets` insert for `scene-files` (private), and 4 `storage.objects`
  policies scoped to `bucket_id = 'scene-files' and (storage.foldername(name))[1] = auth.uid()::text`.
- `excalidraw-app/.env.example` (or repo-root, match where other examples live) — document
  `VITE_APP_SUPABASE_URL=`, `VITE_APP_SUPABASE_ANON_KEY=`, `VITE_APP_FEATURE_SUPABASE_SYNC=true` with comments
  (anon key only — never the service-role key in the client).
- `supabase/README.md` — short setup steps: create project, run `supabase db push` (or paste the migration in
  the SQL editor), confirm the `scene-files` bucket exists, set the 3 env vars, enable email auth (magic link)
  in the Supabase Auth settings.
