# T04 ‚Äî Supabase file adapter (Storage) (+ test)

## Description
Implement the Supabase Storage‚Äìbacked file callbacks consumed by `new FileManager({...})`, matching the real
FileManager injected contract exactly.

## Files
- `excalidraw-app/data/supabase/supabaseFiles.ts`:
  - `createSupabaseFileCallbacks(client, userId) => { getFiles, saveFiles }` where:
    - `getFiles(ids: FileId[]) => Promise<{ loadedFiles: BinaryFileData[]; erroredFiles: Map<FileId, true> }>`
    - `saveFiles({ addedFiles }: { addedFiles: Map<FileId, BinaryFileData> }) => Promise<{ savedFiles: Map<FileId, BinaryFileData>; erroredFiles: Map<FileId, BinaryFileData> }>`
  - `createSupabaseFileManager(client, userId) => FileManager` ‚Äî `new FileManager({ onFileStatusChange: FileStatusStore.updateStatuses.bind(FileStatusStore), ...createSupabaseFileCallbacks(client, userId) })`.
  - Path convention `${userId}/${fileId}`; upload via `client.storage.from('scene-files').upload(path, blob, { upsert:true, contentType })`; download via `.download(path)`. Include the small dataURL‚ÜîBlob util (no shared one fits). Reconstruct `BinaryFileData` fields `{ id, mimeType, dataURL, created, lastRetrieved }` on download.
- `excalidraw-app/tests/supabase/supabaseFiles.test.ts` ‚Äî see acceptance.

## Acceptance criteria
- `saveFiles` uploads each added file to `${userId}/${fileId}` and returns it in `savedFiles`; an upload error
  puts the file into `erroredFiles` as `Map<FileId, BinaryFileData>` (NOT `true`).
- `getFiles` downloads + reconstructs `BinaryFileData` with all required fields; a download error puts the id
  into `erroredFiles` as `Map<FileId, true>`.
- `createSupabaseFileManager` returns a real `FileManager` wired with `onFileStatusChange`.
- Tests pass using the mocked storage client (no network).
