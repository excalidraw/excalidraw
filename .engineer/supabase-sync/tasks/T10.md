## Attempt 2 ŌĆö BUG found by T11, Self-Correction (orchestrator)
**Failure:** T11 found that with VITE_APP_FEATURE_SUPABASE_SYNC="true" the app hangs on the loading spinner forever ŌĆö `isLoading` never clears.
**Root cause (verified by orchestrator reading App.tsx:553-556):** the scene-init effect early-returns when `(!isCollabDisabled && !collabAPI)`. T10 unmounts `<Collab>` under the flag (App.tsx:1111), and `collabAPIAtom` is set ONLY inside `<Collab>` (Collab.tsx:243), so `collabAPI` is permanently null. In a normal (non-iframe) context `isCollabDisabled=false`, so the guard `(true && true)` returns early ŌåÆ `initializeScene` never runs ŌåÆ `initialStatePromiseRef` never resolves ŌåÆ spinner forever. collab.test.tsx misses it (flag-off mounts Collab).
**Fix (orchestrator, 1-line, in T10's domain since T10's unmount caused it):** add `&& !isSupabaseSyncEnabled()` to the guard so the effect proceeds under the flag (initializeScene already handles the flag-on no-external-scene path):
`if (!excalidrawAPI || (!isCollabDisabled && !collabAPI && !isSupabaseSyncEnabled())) return;`
